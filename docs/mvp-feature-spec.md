## 2. 데이터 소스 확정

### 2.1 위성지도 타일

| 항목 | 확정안 |
|------|--------|
| 서비스 | Mapbox Satellite Streets (v12) |
| 타일 URL | `mapbox://styles/mapbox/satellite-streets-v12` |
| API Key | 환경변수 `NEXT_PUBLIC_MAPBOX_TOKEN` |
| 무료 한도 | 50,000 map loads/월 (MVP 충분) |
| 폴백 | 없음 (Mapbox 단일 소스) |

### 2.2 POI 데이터 소스

| 카테고리 | 데이터 소스 | API/파일 | 수집 기준 |
|----------|------------|----------|-----------|
| **지하철역** | 공공데이터포털 — 전국 도시철도역사 정보 | REST API (ServiceKey 필요) | 분석 반경 내 역사. 노선번호·역명·좌표 |
| **학교** | 공공데이터포털 — 전국 학교 기본정보 | REST API | 분석 반경 내. 학교명·학교급(초/중/고)·좌표 |
| **공원** | 공공데이터포털 — 전국 도시공원 정보 | REST API | 분석 반경 내. 공원명·유형·면적·좌표(폴리곤 가용 시 포함) |
| **산** | 국토지리정보원 — 산 위치 정보 | GeoJSON 정적 파일 | 분석 반경 내. 산명·표고·좌표 |
| **아파트 분양** | 공공데이터포털 — 분양가 공시 + 분양 일정 | REST API | 사용자 지정 반경 내. 단지명·세대수·분양가·좌표·분양시기 |

> **MVP 우회 전략:** 공공데이터 API 응답 지연 또는 키 미확보 시, 샘플 대상지(강남역 2km) 범위의 **더미 JSON 시드 데이터**를 `/public/data/` 에 내장하여 개발 진행. API 연동은 Phase 3 이후 순차 교체.

### 2.3 더미 데이터 시드 파일 규격

각 카테고리별 JSON 파일을 `/public/data/seed/` 하위에 배치:

```
public/data/seed/
  subway-stations.json    # 강남역 2km 반경 지하철역
  schools.json            # 강남역 2km 반경 학교
  parks.json              # 강남역 2km 반경 공원
  mountains.json          # 강남역 2km 반경 산
  apartments.json         # 강남역 2km 반경 분양 아파트
```

**공통 필드:**

```typescript
interface POIBase {
  id: string
  name: string
  lat: number
  lng: number
  category: 'subway' | 'school' | 'park' | 'mountain' | 'apartment'
}
```

**카테고리별 확장 필드:**

| 카테고리 | 추가 필드 |
|----------|-----------|
| subway | `line: string` (노선명), `lineColor: string` (노선 HEX 색상) |
| school | `level: 'elementary' \| 'middle' \| 'high'` |
| park | `area_sqm: number`, `polygon?: [number, number][]` |
| mountain | `elevation_m: number` |
| apartment | `units: number` (세대수), `price_per_pyeong: number` (만원), `sale_date: string`, `distance_m: number` (대상지까지 거리) |

## 3. PPT 슬라이드 구성안

**포맷:** 16:9 와이드스크린 (25.4cm x 19.05cm)
**지도 캡처 해상도:** 최소 1920x1080px (devicePixelRatio 2x)

### 슬라이드 순서

| # | 슬라이드 | 콘텐츠 | 지도 레이어 ON |
|---|----------|--------|---------------|
| 1 | **표지** | 프로젝트명, 대상지 주소, 작성일, 회사 로고 영역 | 없음 (배경 이미지만) |
| 2 | **위치 현황** | 위성지도 전체 + 대상지 핀 + 분석 반경 원 | 기본 지도만 |
| 3 | **교통 분석** | 지하철역 마커 + 노선명 라벨 + 대상지 거리선 | subway |
| 4 | **교육 환경** | 학교 마커 (초/중/고 구분) + 학교명 라벨 | school |
| 5 | **자연 환경** | 공원 폴리곤/마커 + 산 마커 + 명칭 라벨 | park, mountain |
| 6 | **분양 현황** | 아파트 마커 + 단지명·세대수·분양가 라벨 + 거리선 | apartment |
| 7 | **종합 분석** | 전체 POI 오버레이 + 요약 텍스트 박스 | 전체 |

### 슬라이드 디자인 규칙

- **지도 영역:** 슬라이드 너비 70% (좌측), 높이 100%
- **정보 패널:** 슬라이드 너비 30% (우측), 카테고리별 범례 + 목록
- **표지:** 중앙 정렬, 위성지도 블러 배경
- **폰트:** 제목 24pt Bold, 본문 14pt Regular, 라벨 11pt
- **여백:** 상하좌우 1.5cm

## 4. 지도 오버레이 표현 규칙

### 4.1 카테고리별 색상 팔레트

| 카테고리 | 메인 컬러 | HEX | 마커 형태 | 라벨 스타일 |
|----------|----------|-----|----------|------------|
| 지하철 | Blue | `#2196F3` | 원형 아이콘 (노선색 테두리) | 흰 배경 박스 + 역명 |
| 학교 | Green | `#4CAF50` | 삼각 깃발 아이콘 (초록) | 흰 배경 박스 + 학교명 |
| 공원 | Light Green | `#66BB6A` | 폴리곤 채움 (20% 투명도) | 중앙 라벨 |
| 산 | Brown | `#8D6E63` | 산 아이콘 (삼각형) | 흰 배경 박스 + 산명 |
| 아파트 | Orange | `#FF7043` | 건물 아이콘 | 흰 배경 박스 + 단지명·세대수 |
| 대상지 | Red | `#E53935` | 핀 마커 (크기 1.5배) | 대상지명 Bold |
| 분석 반경 | Gray | `#9E9E9E` | 점선 원 (stroke-dasharray) | 반경 거리 라벨 |

### 4.2 라벨 가독성 규칙

위성지도 배경 위 텍스트 가독성 확보:

- **텍스트 배경:** 반투명 흰색 박스 (`rgba(255,255,255,0.85)`, border-radius 4px)
- **텍스트 색상:** `#212121` (거의 검정)
- **텍스트 외곽선:** 없음 (박스 배경으로 대체)
- **폰트 크기:** 마커 라벨 12px, 팝업 제목 14px Bold
- **최소 간격:** 마커 간 최소 20px (겹침 방지 — 줌 레벨에 따라 자동 클러스터링)

### 4.3 거리선 표현

대상지 ↔ 주요 POI 직선 거리 표시:

- **선 스타일:** 점선 (`dashArray: [6, 4]`)
- **선 색상:** 해당 카테고리 메인 컬러 (50% 투명도)
- **거리 라벨:** 선 중앙에 `XXm` 또는 `X.Xkm` 표시
- **적용 대상:** 지하철역 (전체), 아파트 (전체)

## 5. 사용자 흐름 (User Flow)

```
[웹 접속]
    ↓
[주소 입력] ← Mapbox Geocoding 자동완성
    ↓
[지도 표시] ← 위성지도 + 대상지 핀 + 2km 반경 원
    ↓
[POI 자동 로드] ← 반경 내 지하철/학교/공원/산 표시
    ↓
[카테고리 토글] ← 사이드바에서 ON/OFF 전환
    ↓
[아파트 표시] ← 반경 내 분양 아파트 자동 로드 + 수동 추가
    ↓
[PPT 미리보기] ← 7장 슬라이드 썸네일 확인 (선택사항)
    ↓
[PPT 다운로드] ← "PPT 생성" 버튼 → .pptx 파일 다운로드
```

## 6. MVP 완료 기준 (Acceptance Criteria)

| # | 기준 | 검증 방법 |
|---|------|-----------|
| AC-1 | 주소 입력 시 위성지도가 해당 위치 중심으로 3초 내 렌더링 | 강남역 주소 입력 → 지도 표시 시간 측정 |
| AC-2 | 분석 반경(1/2/3km) 변경 시 원이 즉시 업데이트 | 드롭다운 변경 → 원 크기 변화 확인 |
| AC-3 | 반경 내 POI 4종(지하철/학교/공원/산)이 올바른 아이콘과 색상으로 표시 | 시드 데이터 기준 마커 수·색상 대조 |
| AC-4 | 반경 내 분양 아파트가 마커+라벨+거리선과 함께 표시 | 시드 데이터 기준 확인 |
| AC-5 | 카테고리별 토글 ON/OFF 시 해당 레이어만 즉시 표시/숨김 | 각 토글 조작 → 지도 변화 확인 |
| AC-6 | PPT 다운로드 시 7장 슬라이드가 올바른 순서·내용으로 생성 | 다운로드 후 PPT 열어 슬라이드별 확인 |
| AC-7 | PPT 지도 이미지 해상도 1920x1080 이상 | 이미지 속성에서 해상도 확인 |
| AC-8 | PPT 내 텍스트(프로젝트명/주소/날짜)가 정확히 반영 | PPT 텍스트 대조 |
| AC-9 | 전체 E2E 흐름(주소 입력→PPT 다운로드)이 60초 내 완료 | 타이머 측정 |
