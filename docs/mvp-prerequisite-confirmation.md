## 1. 위성지도 타일 서비스 확정

### 비교표

| 항목 | Mapbox Satellite | Google Maps | Kakao Map |
|------|-----------------|-------------|-----------|
| 위성타일 해상도 | 최대 zoom 22 (0.5m/px) | 최대 zoom 21 (1m/px) | 최대 zoom 19 (4m/px) |
| 커스텀 오버레이 | GL JS 네이티브 레이어 지원 | Limited (OverlayView) | 제한적 |
| 이미지 캡처 | `map.getCanvas().toDataURL()` 네이티브 | Static Maps API 별도 과금 | 캡처 API 없음 |
| 무료 티어 | 50,000 loads/월 | $200 크레딧/월 | 300,000 loads/일 |
| PPT용 고해상도 캡처 | html-to-image + preserveDrawingBuffer | 별도 Static API 필요 | 불가 |
| 한국 위성 품질 | 양호 (DigitalGlobe) | 양호 (자체 촬영) | 보통 (해상도 낮음) |

### 확정: **Mapbox GL JS 3.x**

**근거:**
1. 커스텀 레이어(GeoJSON, 마커, 폴리곤) 네이티브 지원 — POI 오버레이에 최적
2. Canvas 직접 캡처 가능 — PPT 이미지 생성 워크플로우와 완벽 호환
3. 무료 티어 50,000 loads/월 — MVP 검증 단계 충분
4. 위성 해상도 최고 (zoom 22)

**필요 환경변수:**
```
NEXT_PUBLIC_MAPBOX_ACCESS_TOKEN=<발급 필요>
```

**발급 절차:** mapbox.com 가입 → Dashboard → Access tokens → Default public token 사용

## 2. POI 데이터 출처·갱신주기·정확도

### 데이터 소스 확정표

| POI 카테고리 | 데이터 출처 | 갱신 주기 | 정확도 기준 | MVP 방식 |
|-------------|-----------|----------|-----------|---------|
| 지하철역 | 서울교통공사 역사 좌표 (공공데이터포털) | 연 1회 (노선 변경 시) | 좌표 오차 ≤50m | 시드 데이터 JSON 사용 |
| 학교 | 교육부 학교알리미 API | 연 1회 (3월 학기초) | 좌표 오차 ≤30m | 시드 데이터 JSON 사용 |
| 공원 | 국토교통부 도시공원 정보 | 연 1회 | 면적 오차 ≤5% | 시드 데이터 JSON 사용 |
| 산 | 국토지리정보원 지형도 | 비정기 (지형 변동 시) | 표고 오차 ≤10m | 시드 데이터 JSON 사용 |
| 분양아파트 | 국토교통부 실거래가 공개시스템 | 월 1회 | 거래가 정확, 좌표는 지번 기반 | 시드 데이터 JSON 사용 |

### MVP 데이터 전략

**Phase 1 (MVP):** 시드 데이터(JSON) 기반 — API 연동 없이 즉시 개발 가능
- 강남역 2km 반경 시드 데이터: 완비 (6역, 6교, 4공원, 2산, 5아파트)
- 청와대 3km 반경 시드 데이터: 완비 (28역, 학교, 공원, 산, 5+아파트)

**Phase 2 (확장):** 공공데이터포털 API 직접 연동
- API 키 발급: data.go.kr 회원가입 후 각 API별 활용신청
- 응답 포맷: JSON (XML도 가능하나 JSON 권장)

### 정확도 수용 기준

| 항목 | 허용 오차 | 검증 방법 |
|------|----------|----------|
| 좌표 위치 | ≤50m (지도 위 육안 확인 가능) | 위성지도 위 마커 위치 확인 |
| 거리 계산 | ≤100m (Haversine 공식) | 기준점~POI 직선거리 |
| 면적 (공원) | ≤10% | 공공데이터 대조 |
| 가격 정보 | 실거래가 원본 그대로 사용 | 출처 명시 |

## 3. 사용자 지정 범위 입력 방식

### 비교표

| 방식 | 장점 | 단점 | 구현 난이도 |
|------|------|------|-----------|
| A. 주소 검색 + 반경(km) 슬라이더 | 직관적, 빠름 | 원형 범위만 가능 | 낮음 |
| B. 주소 검색 + 지도 위 드래그 원 | 시각적 피드백 | 터치 이벤트 복잡 | 중간 |
| C. 좌표 직접 입력 + 반경 | 정밀함 | 일반 사용자 접근성 낮음 | 낮음 |
| D. 지도 위 다각형 그리기 | 자유로운 범위 지정 | 구현 복잡, PPT 표현 어려움 | 높음 |

### 확정: **방식 A (주소 검색 + 반경 슬라이더)**

**근거:**
1. MVP 목적에 최적 — 빠른 구현, 직관적 UX
2. Mapbox Geocoding API로 주소→좌표 변환 즉시 가능
3. 반경 슬라이더 (1km~5km, 기본값 2km) — 원형 범위 시각화 단순
4. 향후 방식 B, D로 점진적 확장 가능

**입력 플로우:**
```
[주소 검색 입력] → [자동완성 드롭다운] → [선택] → [지도 이동]
         ↓
[반경 슬라이더: 1km ──●── 5km] → [원형 범위 표시] → [POI 자동 로딩]
```

**기본값:**
- 검색 입력: 빈 상태 (플레이스홀더: "분석할 위치를 검색하세요")
- 반경: 2km
- 슬라이더 단위: 0.5km

## 4. 보고서형 아이콘·선·도형 규격

| POI | 아이콘 | 크기 | 색상 | 파일 |

| 산 | 삼각형 | 24px / 48px | #8D6E63 | mountain.svg |

> SVG 아이콘 초안 5종은 디자인팀 워크트리(5ed9a203)에서 제작 완료

### 선·도형 규격

| 요소 | 스타일 | 색상 | 투명도 |
|------|-------|------|-------|
| 반경 원 | 대시 점선 (dasharray: 8,4) | #9E9E9E | 30% fill |
| 거리선 (아파트→기준점) | 실선 2px | #FF7043 | 80% |
| 공원 영역 | 폴리곤 채우기 | #66BB6A | 20% fill, 실선 border |
| 지하철 노선 | 실선 3px | 각 노선 고유색 | 60% |

### 라벨 규격

| 항목 | 폰트 | 크기 | 배경 | 색상 |
|------|------|------|------|------|
| POI 이름 | Pretendard (한), Inter (영) | 12px | rgba(255,255,255,0.85) box | #212121 |
| 팝업 정보 | Pretendard | 14px | 흰색 카드 (radius 8px, shadow) | #212121 |
| 거리 라벨 | Inter (숫자) | 11px | rgba(255,255,255,0.9) | #FF7043 |
| 가격 라벨 | Inter (숫자) | 13px bold | rgba(255,255,255,0.9) | #E53935 |

### 마커 간격 규칙
- 최소 간격: 20px — 겹칠 경우 자동 클러스터링
- 클러스터 표시: 원형 + 개수 숫자 (예: "3")
- 줌 레벨별 표시: zoom ≥14에서 개별 마커, zoom <14에서 클러스터

## 5. 개발팀 즉시 착수 체크리스트

### 블로킹 항목 (착수 전 필수)

| # | 항목 | 상태 | 담당 | 비고 |
|---|------|------|------|------|
| 1 | Mapbox Access Token 발급 | **미완** | 개발팀 | mapbox.com 가입 후 즉시 발급 가능 (5분) |

### 비블로킹 항목 (시드 데이터로 우회 가능)

| # | 항목 | 상태 | 담당 | 비고 |
|---|------|------|------|------|
| 2 | 공공데이터포털 API 키 | 시드 데이터로 대체 | 기획팀 | Phase 2에서 연동 |
| 3 | SVG 아이콘 최종본 | 초안 완료 (5ed9a203) | 디자인팀 | 타일 해상도 확정 후 보정 |
| 4 | 폰트 라이선스 확인 | **미완** | 기획팀 | Pretendard(OFL), Inter(OFL) — 무료 |

### 개발 순서 (확정)

```
Day 1:  프로젝트 세팅 (Next.js + Mapbox + Tailwind + Zustand + shadcn/ui)
Day 2-3: 주소 검색 + 위성지도 렌더링 + 반경 원 표시 + 캡처
Day 4-5: POI 오버레이 4종 (시드 데이터 기반) + 토글 + 팝업
Day 6-7: 분양아파트 마커 + 거리선 + 가격 라벨 + 수동 추가 폼
Day 8-9: PptxGenJS 7슬라이드 생성 + 템플릿
Day 10:  QA + 에러 처리 + 폴리시
```

### 수락 기준

1. 주소 입력 → 위성지도 렌더링 ≤3초
2. 4종 POI 마커 정상 표시 (반경 내)
3. 분양아파트 거리선 + 가격 라벨 표시
4. 7슬라이드 PPT 다운로드 ≤10초
5. PPT 이미지 해상도 ≥1920x1080px

## 부록: 시드 데이터 스키마 요약

```typescript
// 공통 필드
interface BasePOI {
  id: string
  name: string
  lat: number
  lng: number
}

// 지하철
interface SubwayStation extends BasePOI {
  line: string        // "2호선"
  lineColor: string   // "#33A23D"
}

// 학교
interface School extends BasePOI {
  level: 'elementary' | 'middle' | 'high'
}

// 공원
interface Park extends BasePOI {
  area_sqm: number
  polygon?: [number, number][]
}

// 산
interface Mountain extends BasePOI {
  elevation_m: number
}

// 분양아파트
interface Apartment extends BasePOI {
  units: number
  price_per_pyeong: number  // 만원
  sale_date: string         // "YYYY-MM"
  distance_m: number        // 기준점으로부터
}
```
