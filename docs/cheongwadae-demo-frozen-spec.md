## 2. 지도 엔진 확정

### 2.1 선정: Mapbox GL JS 3.x

| 기준 | Mapbox GL JS | Kakao Maps | Naver Maps |
|------|-------------|------------|------------|
| 위성 타일 | mapbox://styles/mapbox/satellite-v9 | 제한적 | Static API만 |
| 커스텀 오버레이 | Canvas 기반 자유 렌더링 | HTML DOM 오버레이 | 제한적 |
| 캔버스 캡처 | `preserveDrawingBuffer: true` 네이티브 | 불가 | Static API 캡처만 |
| 무료 티어 | 50K loads/월 | 300K/일 | 월별 제한 |
| PPT 삽입 호환성 | PNG 캡처 → PptxGenJS 직삽입 | 추가 변환 필요 | 추가 변환 필요 |

**확정 사유:** 위성 타일 품질 + Canvas 네이티브 캡처 + 커스텀 레이어가 PPT 생성 파이프라인에 가장 적합

### 2.2 위성지도 타일 설정

```
Style: mapbox://styles/mapbox/satellite-streets-v12
Zoom level: 13 (3km 반경 최적)
Center: [126.9748, 37.5866]
Canvas size: 1920 x 1080px (16:9)
preserveDrawingBuffer: true
```

### 2.3 API 키 관리 (인프라보안팀 요구사항)

| 항목 | 규격 |
|------|------|
| 저장 위치 | `.env.local` (`NEXT_PUBLIC_MAPBOX_TOKEN`) |

| 커밋 이력 노출 금지 | 커밋 전 `git diff --cached`로 시크릿 포함 여부 검증 |

## 3. POI 데이터 소스 스키마

### 3.1 공통 인터페이스

```typescript
interface PoiBase {
  readonly id: string;        // "{category_prefix}-{3자리 번호}" (예: sub-001)
  readonly name: string;      // 한글 명칭
  readonly lat: number;       // WGS84 위도
  readonly lng: number;       // WGS84 경도
  readonly category: PoiCategory;
}

type PoiCategory = "subway" | "school" | "park" | "mountain" | "apartment";
```

### 3.2 카테고리별 확장 필드

| 카테고리 | 확장 필드 | 타입 | 설명 |
|----------|----------|------|------|
| subway | `line` | string | 호선명 (예: "3호선") |
| subway | `lineColor` | string | 서울교통공사 공식 노선색 HEX |
| school | `level` | "elementary" \| "middle" \| "high" | 학교급 |
| park | `area_sqm` | number | 면적 (㎡) |
| park | `type` | string | 유형 (근린공원/사적/역사공원 등) |
| mountain | `elevation_m` | number | 해발 고도 (m) |
| apartment | `units` | number | 총 세대수 |
| apartment | `price_per_pyeong` | number | 분양가 (만원/평) |
| apartment | `sale_date` | string | 분양 시기 (YYYY-MM) |
| apartment | `distance_m` | number | 중심점까지 직선거리 (m) |

### 3.3 시드 데이터 파일 구조

```
public/data/seed/cheongwadae/
  ├── subway-stations.json   (28개 역)
  ├── schools.json           (24개 학교)
  ├── parks.json             (13개 공원/궁궐)
  ├── mountains.json         (5개 산)
  └── apartments.json        (8개 분양단지)
```

### 3.4 데이터 수집 기준

| 카테고리 | 수집 범위 | 출처 | 좌표 오차 허용 |
|----------|----------|------|---------------|
| 지하철 | 중심 3km 이내 모든 역 | 서울교통공사 역사 좌표 | ≤ 50m |
| 학교 | 중심 3km 이내 초/중/고 | 학교알리미 (schoolinfo.go.kr) | ≤ 50m |
| 공원 | 중심 3km 이내 주요 공원/궁궐 | 서울시 공원녹지 데이터 | ≤ 100m |
| 산 | 중심 3km 이내 주요 산 | 국토지리정보원 | ≤ 100m |
| 아파트 | 중심 3km 이내 최근 분양 | 국토교통부 분양가공시 | ≤ 50m |

### 3.5 누락 판정 기준

- 지하철: 출구 기준 중심점 직선 3km 초과 시 제외
- 학교: 정문 기준 3km 초과 시 제외
- 공원: 중심점(공원 대표좌표) 기준 3km 초과 시 제외
- 산: 정상 좌표 기준 3km 초과 시 제외 (단 주요 산은 3.5km까지 허용)
- 아파트: 단지 대표 좌표 기준 3km 초과 시 제외

## 5. PPT 생성 파이프라인

### 5.1 생성 방식: 클라이언트 사이드 (PptxGenJS 3.x)

```
[사용자 입력] → [Mapbox 렌더링] → [html-to-image 캡처 (1920x1080, 2x DPR)]
     → [PptxGenJS 슬라이드 생성] → [.pptx 다운로드]
```

**서버 불필요:** 브라우저 내에서 전체 파이프라인 실행

### 5.2 슬라이드 구성 (7슬라이드)

| # | 슬라이드 | 콘텐츠 | 레이아웃 |
|---|---------|--------|---------|
| 1 | 표지 | 프로젝트명, 분석 위치, 반경, 날짜 | 네이비 배경, 중앙 정렬 |
| 2 | 위치 개요 | 위성지도 전체뷰 + 전체 POI + 범례 | 지도 70% (좌) + 패널 30% (우) |
| 3 | 교통 분석 | 지하철역만 표시 + 노선별 분류표 | 지도 70% + 노선표 30% |
| 4 | 교육 환경 | 학교만 표시 + 학교급별 분류표 | 지도 70% + 분류표 30% |
| 5 | 자연 환경 | 산 + 공원 표시 + 면적/고도 정보 | 지도 70% + 정보표 30% |
| 6 | 분양 아파트 | 아파트 + 커넥터선 + 상세 정보 카드 | 지도 70% + 데이터 테이블 30% |
| 7 | 종합 요약 | 카테고리별 집계 + 핵심 수치 | 네이비 배경, 요약 카드 레이아웃 |

### 5.3 슬라이드 마스터 규격 (디자인팀 레이아웃 그리드)

```
슬라이드 크기: 13.333" × 7.5" (16:9, 1920×1080px)

그리드: 12컬럼 (각 1.028"), 거터 0.139"

지도 영역: x=0.3", y=0.3", w=9.333", h=6.9"
우측 패널: x=9.733", y=0.3", w=3.3", h=6.9"

타이틀바: x=0.3", y=0.3", w=9.333", h=0.6"
  - 배경: rgba(11,34,57,0.85)
  - 텍스트: 맑은 고딕 Bold, 16pt, #FFFFFF

범례 패널: x=9.733", y=0.3", w=3.3", h=auto
  - 배경: rgba(255,255,255,0.92)
  - 라운드: 4pt
  - 패딩: 0.15"
  - 드롭섀도: blur 4px, rgba(0,0,0,0.2)

푸터: x=0.3", y=7.05", w=12.733", h=0.3"
  - 텍스트: 맑은 고딕, 7pt, #999999
  - 내용: "자동 생성 보고서 | 데이터 기준일: {date} | 좌표계: WGS84"
```

## 6. 데모 시나리오

### 6.1 실행 시나리오 (대표님 시연용)

```
1. 앱 접속 (localhost:3000)
2. 지도에 "청와대" 위치가 중심에 표시됨 (기본 프리셋)
3. 반경 3km 점선 원이 오버레이됨
4. 사이드바에서 레이어 토글:
   a. 지하철 ON → 28개 역 아이콘 표시 (노선색 구분)
   b. 학교 ON → 24개 학교 아이콘 표시 (급별 구분)
   c. 공원 ON → 13개 공원/궁궐 아이콘 표시
   d. 산 ON → 5개 산 삼각형 아이콘 표시
   e. 아파트 ON → 8개 분양단지 + 커넥터선 + 상세 라벨
5. "PPT 다운로드" 버튼 클릭
6. 7슬라이드 .pptx 파일 자동 다운로드 (10초 이내)
7. PPT 열어서 각 슬라이드 확인
```

### 6.2 기대 결과물

| 검증 항목 | 기대값 |
|----------|--------|
| 파일 형식 | `.pptx` (Microsoft PowerPoint 호환) |
| 슬라이드 수 | 7장 |
| 파일 크기 | 2~8MB (지도 이미지 해상도에 따라) |
| 슬라이드 비율 | 16:9 (1920×1080) |
| 위성지도 포함 | 각 슬라이드 배경에 위성지도 캡처 이미지 |
| POI 마커 총 수 | 78개 (28+24+13+5+8) |
| 아파트 상세 정보 | 단지명, 세대수, 평당가, 분양시기, 거리 |
